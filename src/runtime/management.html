Running on git-azure by Tomasz Janczuk

Real time logs

Real time logging delivers stdout and stderr output from your applications as well as git-azure system messages to the browser or the terminal window using WebSockets.

Access real time logs in the browser now

Access real time logs in the terminal with: 

git azure logs

Soft reset

Soft reset of the git-azure runtime recycles all application processes, syncs the Git repository, and updates system configuration. The actions are identical to the actions performed when a post receive hook URL is invoked from Git on git push. The soft reset takes seconds. Active logging sessions remain active throughout the soft reset. Application endpoints are not available during the soft reset.

Initiatie soft reset now

Initiate soft reset from the terminal with: 

git azure reset

Hard reset

Hard reset of the git-azure runtime recycles the git-azure reverse proxy as well as all applications, syncs the Git repository, and updates system configuration. Hard reset restarts the git-azure runtime and is useful when an update to the git-azure runtime code in the .git-azure submodule has been checked in. Hard reset typically takes well under a minute. Management and application endpoints are unavailable during that time.

Initiatie hard reset now

Initiate hard reset from the terminal with: 

git azure reset --hard
